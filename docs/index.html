Disposable browser

ITERA

A browser identity for one session only. When the window closes, the identity dies. No past survives.

View on GitHub Build locally

A browser without continuity.

Itera does not try to be a normal browser with a privacy mode. It treats the entire browser identity as disposable runtime state.

Born

Every launch creates a fresh browser identity with a temporary profile.

Used

The session behaves like a normal browser window while it is alive.

Forgotten

Closing Itera destroys the temporary profile and removes abandoned sessions on the next launch.

Session death is the feature.

01

03

Future modes.

Itera is structured to grow toward RAM-only sessions, per-site disposable identities, optional routing modes, and a session monitor. Those are future features. The core loop comes first.
