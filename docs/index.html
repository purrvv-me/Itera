ITERA

A browser identity that exists for one session only.

The idea

Not a private browser. A disposable one.

Modern browsers remember the user. Private modes try to limit what is saved. Itera changes the unit of trust: the whole browser identity is temporary.

The goal is not "my history was cleared." The goal is "my previous browser identity no longer exists."

Product shape

Designed to feel like a real browser, without keeping a past.

Itera has tabs, a start page, browser controls, and a standalone app shell. Those features make the disposable model usable without changing the rule: one launch equals one life.

1

One temporary identity

Every app launch creates a new session boundary for browsing state.

2

Tabs inside the session

Multiple tabs are supported, but they belong to the current identity only.

3

No continuity layer

No sync, accounts, bookmarks, session restore, or recently closed tabs.

Interface preview

The browser surface is part of the product.

Itera should not feel like a script that opens a webview. The visible shell communicates the model: a temporary browser identity, tabs inside one launch, and a clean session end.

Core feature

Session death is not cleanup. It is the product.

Itera does not position itself as another privacy toggle. The product is a browser identity that is born, used, and destroyed.

Positioning

What Itera is, and what it is not.

Itera is

  • A one-time-use browser identity.
  • A session-based browser runtime.
  • A browser without continuity between launches.
  • A product built around identity death.

Itera is not

  • Not a Tor replacement.
  • Not a VPN.
  • Not a tracker blocker pitch.
  • Not a normal incognito mode with branding.
