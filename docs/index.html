Not incognito. Not a private mode.

Itera is built around continuity deletion. It does not ask the browser to forget history later. It creates a temporary browser identity, then removes the entire identity when the session ends.

No past survives

Close the window and Itera destroys the temporary session profile.

Core product idea

Session death is the feature.

The key feature is not tabs, bookmarks, sync, themes, or accounts. The key feature is that the previous browser identity no longer exists.

What disappears.

The disposable profile is intended to remove session state such as cookies, cache, local storage, permissions, saved form state, and browser profile files.

State

Cookies and cache

Temporary browser data belongs to the current session only.

Storage

Local app data

localStorage, IndexedDB, permissions, and profile state are part of the disposable identity.

Recovery

Abandoned sessions

Old Itera session folders are cleaned on launch if a previous process was interrupted.
