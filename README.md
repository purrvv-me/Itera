# ITERA

**Every launch begins again.**

ITERA is a disposable-identity browser for Windows. Every launch is a brand-new
user for the internet: no saved history, no cookies, no continuity. When you
close it, a terminal window appears and **visibly erases** the session from disk
— so you can watch your tracks disappear.

## Download

Grab the latest Windows installer from the **[Releases](../../releases/latest)** page,
or visit the website (GitHub Pages, served from [`docs/`](docs/)).

> The site auto-links to this repo's releases when hosted on GitHub Pages. For a
> local preview, edit `REPO_FALLBACK` in `docs/index.html`.

## Releasing (automated)

Pushing a version tag builds the installer on GitHub's Windows runners and
publishes it to a Release automatically (`.github/workflows/release.yml`):

```bash
git tag v1.0.0
git push origin v1.0.0
```

The workflow runs `electron-builder`, then attaches `ITERA-Setup-<version>.exe`
to a new GitHub Release. The website's download button points at
`releases/latest`, so it picks up each new build with no changes. You can also
trigger the workflow manually from the **Actions** tab to test a build without
releasing.

## How it works

- **Fresh identity per launch.** Each run gets its own unique, on-disk session
  partition (`Partitions/itera-<id>`), so the browser always starts empty.
- **Randomized user agent.** A realistic desktop-Chrome UA is generated every
  session (`src/main.js → randomUserAgent`).
- **Visible wipe on exit.** On close, ITERA clears all storage via Electron's
  APIs, then spawns `scripts/cleanup.bat` in a real terminal that deletes the
  session folder in front of you (cookies, cache, localStorage, sessionStorage,
  fingerprint data).
- **Stale-session purge.** If a previous run crashed, leftover ITERA partitions
  are deleted on the next startup.

## Run it

```powershell
npm install
npm start
```

## Package a Windows installer

```powershell
npm run dist
```

Output goes to `dist/`. The icon is generated by `scripts/make-icon.ps1`.

## Update the installed app (no reinstall)

Once ITERA is installed, you don't need to run the installer again after code
changes. Run:

```powershell
npm run update
```

This rebuilds the app and swaps the new `app.asar` straight into the installed
folder (`%LOCALAPPDATA%\Programs\itera`), closing the app first if it's running.
Takes a couple of seconds. Just relaunch ITERA afterward.

> Note: `cleanup.bat` ships inside `app.asar`. At exit it is copied to a temp
> file and run from there, because `cmd.exe` cannot execute a script from inside
> the asar archive.

## Notes on DRM (Spotify / Netflix)

Standard Electron does not ship the Widevine CDM, so most sites work out of the
box (including YouTube), but **DRM-protected streaming** (Spotify Web Player,
Netflix) needs Widevine. To support it, swap the Electron dependency for the
[castLabs Electron for Content Security (ECS)](https://github.com/castlabs/electron-releases)
build, which bundles and provisions Widevine. No code changes are required.

## Project layout

```
src/main.js              Electron main process — identity, session, wipe
src/preload.js           Safe bridge to the renderer
src/renderer/index.html  Start screen + browser chrome
src/renderer/styles.css  Dark + orange theme
src/renderer/renderer.js UI logic, webview, navigation
scripts/cleanup.bat      The visible deletion terminal
```
